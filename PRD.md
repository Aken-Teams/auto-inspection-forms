# 檢驗表自動核對系統 產品需求文件 (PRD)

## 1. 產品概述

### 產品願景與定位
- 打造一個自動化的製造業點檢表核對平台，讓品質管理人員上傳 Excel 點檢記錄表後，系統自動辨識表單類型、解析數據、比對規格標準，產出 OK/NG 判定結果並匯出標註報表，大幅減少人工核對時間與錯誤率。
- 定位為工廠品質管理部門的內部工具，聚焦於「規格匯入 → 資料上傳 → 自動判定 → 報表匯出」的端到端自動化流程。

### 商業目標
- 將人工逐筆核對點檢數據的作業時間減少 90% 以上。
- 消除人為疏漏，確保每一筆檢驗數據都經過規格比對。
- 支援零程式碼擴展新表單類型，降低 IT 維護成本。
- 透過規格版本追蹤與回溯，確保判定結果的可追溯性與稽核合規性。

### MVP 範圍定義
- 核心功能為：表單類型自動辨識（6 級優先辨識）、Excel 數據解析（5 種內建 + 通用解析器）、規格匯入與管理（含版本追蹤）、自動判定引擎（OK/NG/SKIP）、匯出標註報表（NG 紅色標註 + 判定欄）。
- 首版支援 5 種內建表單類型（有專用解析器）+ 19 種 AU 表單類型（通用解析器）。
- 首版不含使用者身分驗證與權限控管，所有端點開放存取。
- AI 輔助辨識為可選功能，系統在無 AI 的情況下仍可正常運作。

## 2. 使用者分析

### 目標使用者 Persona
- **品質管理員 (QC)**：負責日常上傳點檢表、檢視判定結果、下載標註報表。需要快速上傳大量檔案並一目了然地看到 OK/NG 結果。
- **規格管理員 (Spec Admin)**：負責維護各表單類型的規格標準，包括匯入規格、編輯判定條件、管理版本歷史。需要預覽匯入變更、偵測重複、回溯歷史版本。
- **工廠主管 (Manager)**：需要從歷史紀錄總覽所有上傳批次的判定結果，快速識別異常批次並下載報表供稽核使用。

### 使用者痛點與需求
- 痛點：目前人工逐筆核對點檢數據，耗時且容易遺漏。
- 痛點：不同設備的規格散落在各種 Excel 匯總表中，查找困難且版本混亂。
- 痛點：新表單類型需要 IT 開發專用程式碼才能支援，擴展緩慢。
- 痛點：匯出報表需手動標註 NG 欄位，效率低下且格式不一致。
- 需求：一鍵上傳即可自動完成辨識、解析、判定、標註的完整流程。
- 需求：規格變更時能預覽差異、保留歷史、隨時回溯。
- 需求：新增表單類型時不需要寫程式，只要上傳範例檔案即可。

### 使用者旅程
1. **規格建立階段**：規格管理員在「規格管理」頁面新增表單類型，上傳含規格的 Excel 檔案。系統自動從匯總 Sheet 或表頭提取檢查項目與規格範圍，產生預覽供確認後匯入。支援 AI 輔助提取（可選）。
2. **日常檢驗階段**：品質管理員在「上傳檢驗」頁面拖放一或多個 Excel 點檢表。系統自動辨識表單類型、解析各 Sheet 數據、比對規格產出 OK/NG 判定。結果即時顯示於頁面，可逐 Sheet 檢視詳細數據與判定明細。
3. **報表匯出階段**：品質管理員下載標註報表（單 Sheet / 整份 / 批次 ZIP）。報表在原始 Excel 上標註 NG 儲存格（紅色）並新增判定欄（OK/NG/—），直接用於稽核歸檔。
4. **歷史查閱階段**：工廠主管在「檢驗履歷」頁面瀏覽所有上傳批次，按時間排序，展開查看各檔案的 OK/NG 統計。可刪除舊批次或重新下載報表。

## 3. User Stories

### 依角色分類的使用者故事

**身為品質管理員 (QC)，我希望能...**
- US-QC-01: 拖放一或多個 Excel 檔案上傳，系統自動辨識表單類型並開始處理。
- US-QC-02: 上傳完成後立即看到各 Sheet 的 OK/NG/未設規格統計結果。
- US-QC-03: 點擊單一 Sheet 查看逐筆數據與對應的規格比對結果（含規格範圍顯示）。
- US-QC-04: 下載單一 Sheet、整份檔案或整批上傳的標註報表（Excel/ZIP）。
- US-QC-05: 在「檢驗履歷」頁面查看所有歷史上傳批次，按時間排序並顯示 OK/NG 統計。
- US-QC-06: 刪除不需要的上傳批次，且刪除後頁面立即更新。

**身為規格管理員 (Spec Admin)，我希望能...**
- US-SA-01: 新增自訂表單類型（代碼、名稱、檔名模式），不需要寫程式碼。
- US-SA-02: 上傳規格 Excel 後，系統自動提取檢查項目並顯示預覽，確認後匯入。
- US-SA-03: 對於沒有「匯總」Sheet 的檔案，系統從資料 Sheet 表頭自動提取規格。
- US-SA-04: 編輯規格項目的判定條件（範圍、勾選、文字、門檻值、略過）。
- US-SA-05: 查看規格版本歷史，了解每次變更的內容，並能回溯至任意歷史版本。
- US-SA-06: 匯入時系統自動偵測重複（檔案雜湊 + 內容級比對），避免重複匯入。
- US-SA-07: 批次建立多個表單類型，從範例檔案自動建立並匯入規格。

**身為工廠主管 (Manager)，我希望能...**
- US-MG-01: 在檢驗履歷頁面一覽所有上傳批次的狀態，快速識別含 NG 的批次。
- US-MG-02: 下載任意批次的完整標註報表供稽核歸檔。
- US-MG-03: 介面支援繁中 / 簡中 / 英文切換，適應不同廠區人員。

### 驗收標準（Acceptance Criteria）
- **US-QC-01 的 AC**：
  - 支援 `.xlsx` 格式的 Excel 檔案上傳。
  - 支援拖放（drag-and-drop）與檔案選擇器兩種方式。
  - 批次上傳時，所有檔案共用同一個 batch_id。
  - 上傳後顯示即時處理進度（pending → processing → completed/error）。
  - 系統透過 6 級優先辨識自動判定表單類型。
- **US-QC-04 的 AC**：
  - 匯出的 Excel 保留原始檔案格式與資料。
  - NG 儲存格標註為紅色底色 + 紅色粗體字。
  - 自動新增「判定」欄，顯示 OK / NG / —（未匹配規格）。
  - 多行記錄（如上模/下模）的判定欄自動合併儲存格。
  - 批次下載為 ZIP 格式，內含各檔案的標註版本。
- **US-SA-02 的 AC**：
  - 預覽介面顯示將新增、修改、刪除的規格項目明細。
  - 顯示重複偵測結果（檔案雜湊重複或內容重複）。
  - 使用者確認後才實際寫入資料庫。
  - 匯入完成後自動建立版本快照。
- **US-QC-05 的 AC**：
  - 批次以卡片形式顯示，含檔案數、Sheet 數、OK/NG/未設規格統計。
  - 左側色條指示狀態：紅色（有 NG）、綠色（全 OK）、灰色（未設規格）。
  - 支援分頁瀏覽（每頁 10 筆）。
  - 展開批次可查看各檔案明細。

## 4. 功能需求（MoSCoW 優先級）

### Must Have：MVP 核心功能

- **表單類型自動辨識** ✅：
  - 6 級優先辨識引擎：檔名正則 → 內建關鍵字 → Sheet 名模式 → Sheet 內容 → 資料庫模式 → AI 辨識。
  - 5 種內建表單類型的專用解析器。
  - 通用解析器（GenericParser）支援任意未知表單。

- **Excel 數據解析** ✅：
  - **標準行列格式**：自動偵測中文表頭行，解析各欄數據。
  - **多層表頭**：偵測子標題行（中文/英文），合併為群組結構。
  - **轉置/樞紐格式**：偵測日期橫向展開的格式，自動轉置為行列結構。
  - 設備 ID 自動提取（從 Sheet 名稱解析設備編號）。
  - 合併儲存格正確處理（讀取與寫入）。

- **規格管理** ✅：
  - 表單類型 CRUD（代碼、名稱、描述、檔名模式、辨識關鍵字）。
  - 規格匯入：從「匯總」Sheet 提取或從表頭自動提取。
  - 匯入預覽：顯示新增/修改/刪除的差異，確認後才寫入。
  - 重複偵測：檔案雜湊（SHA-256）+ 內容級比對。
  - 規格項目編輯：5 種判定類型（range、check、text、threshold、skip）。
  - 版本追蹤：每次變更自動建立版本快照，可查看歷史與回溯。
  - **表頭規格提取**：無匯總 Sheet 時，自動從 data sheet 表頭提取檢查項目與內嵌規格範圍。

- **自動判定引擎** ✅：
  - 四層模糊匹配：精確匹配 → 正規化匹配 → 去單位匹配 → 子字串包含。
  - 5 種規格判定類型：
    - `range`：數值範圍（min ≤ value ≤ max）
    - `check`：勾選確認（√ / OK / 正常）
    - `text`：文字完全匹配
    - `threshold`：門檻值（≥、≤、>、<）
    - `skip`：略過不判定
  - 行級判定：任一欄 NG → 整行 NG；全部 OK → 整行 OK；全部略過 → SKIP。
  - UNIVERSAL 規格 fallback：無設備專屬規格時，使用通用規格。

- **匯出標註報表** ✅：
  - 在原始 Excel 上直接標註，保留原始格式。
  - NG 儲存格：紅色底色 + 紅色粗體字。
  - 自動新增「判定」欄：OK / NG / —（未匹配規格）。
  - 多行記錄判定欄自動垂直合併儲存格。
  - 處理隱藏欄與合併儲存格（自動 unmerge 後寫入）。
  - 匯出時即時重新判定（使用最新規格）。
  - 批次匯出為 ZIP。

- **批次上傳** ✅：
  - 一次上傳多個檔案，共用 batch_id。
  - 各檔案獨立處理、獨立判定。
  - 批次級統計彙總。

- **檢驗履歷** ✅：
  - 批次卡片式瀏覽，含 OK/NG 統計與左側色條狀態指示。
  - 分頁瀏覽（每頁 10 筆）。
  - 展開查看檔案級明細。
  - 批次刪除（含檔案清理），樂觀更新 UI。
  - 單檔或批次下載。
  - SQL 聚合查詢優化（2 條 SQL 取代 N+1）。

- **多語系** ✅：
  - 繁體中文（zh-TW，預設）、簡體中文（zh-CN）、英文（en）。
  - 瀏覽器語言自動偵測 + 手動切換。

- **AI 輔助辨識**（🔧 可選功能）：
  - DeepSeek API 整合，用於表單類型辨識與規格提取。
  - 無 API Key 時系統自動 fallback 到規則引擎，不影響核心功能。

### Should Have：重要但可稍後

- **使用者身分驗證與權限控管**：
  - JWT 或 Session 驗證機制。
  - 角色權限：管理員（可管理規格）、操作員（僅可上傳與查看）。
  - API 層面的存取控制。

- **儀表板與統計分析**：
  - 判定結果趨勢圖（按日/週/月）。
  - 各表單類型的 NG 率統計。
  - 設備級的異常頻率排名。

- **通知機制**：
  - NG 數量超過門檻時自動通知。
  - 規格變更時通知相關人員。

- **自訂判定門檻**：
  - 允許按表單類型或設備自訂 NG 判定條件（如容許偏差百分比）。

- **批次規格匯入腳本**：
  - 一鍵從指定目錄批次註冊所有表單類型並匯入規格。

### Could Have：錦上添花

- **Excel 線上預覽**：
  - 在瀏覽器中直接預覽上傳的 Excel 內容，無需下載。

- **規格範本庫**：
  - 預建常用設備的規格範本，新增表單時可直接套用。

- **Webhook / Email 通知**：
  - 判定完成後透過 Webhook 或 Email 通知相關人員。

- **稽核日誌**：
  - 記錄所有操作（上傳、刪除、規格變更）的時間與操作者。

- **雲端儲存整合**：
  - 支援 AWS S3 / 阿里雲 OSS 儲存上傳檔案與規格檔案。

- **進階 AI 功能**：
  - AI 自動分析 NG 原因並產生改善建議。
  - AI 輔助從非標準格式的 Excel 提取規格。

### Won't Have：明確排除（MVP 範圍外）
- 手機原生應用程式（MVP 以 Web 響應式設計為主）。
- 與 ERP / MES 系統的即時整合。
- 複雜的多層級簽核流程。
- 即時多人協作編輯規格。
- 圖片或 PDF 格式的點檢表辨識（僅支援 Excel）。

## 5. 非功能性需求

### 效能
- 檢驗履歷頁面載入時間應小於 1 秒（已優化為 2 條 SQL 聚合查詢）。
- 單檔上傳處理（辨識 + 解析 + 判定）應在 5 秒內完成。
- 批次上傳 5 個檔案的處理時間應在 15 秒內。
- 匯出標註報表的產生時間應在 3 秒內。
- 批次刪除操作應在 2 秒內完成（使用 bulk delete）。

### 安全性
- 所有傳輸資料使用 HTTPS 加密。
- （📋 規劃中）使用者身分驗證與角色權限控管。目前所有端點開放存取。
- 檔案上傳使用 UUID 前綴儲存，避免檔名衝突與路徑穿越攻擊。
- 規格檔案以 SHA-256 雜湊驗證完整性。
- CORS 設定需在正式部署時限縮為指定來源（目前為 `*`）。

### 可用性
- 介面設計直觀，品質管理員無需培訓即可完成上傳與查看結果。
- 判定結果以顏色直觀區分：綠色 OK、紅色 NG、黃色未設規格。
- 操作回饋即時：上傳進度指示、刪除樂觀更新、Toast 通知。
- 支援繁中/簡中/英文三語系，適應不同廠區。

### 可靠性
- 規格變更自動建立版本快照，確保可追溯與回溯。
- 匯出報表時即時重新判定（使用最新規格），確保結果正確。
- 刪除操作需二次確認（ConfirmDialog）。
- 合併儲存格自動 unmerge 處理，避免寫入失敗。
- 通用解析器能處理 85-90% 的工廠點檢表格式。

## 6. 成功指標（KPI）

### 量化的成功標準
- **處理效率**：相較於人工核對，判定時間減少 90%（原本 30 分鐘/檔 → 3 秒/檔）。
- **覆蓋率**：上線六個月內，工廠 80% 的點檢表類型已建立規格並可自動判定。
- **準確率**：自動判定結果與人工核對的一致率達 99% 以上。
- **使用者採用率**：上線三個月內，目標部門 90% 的點檢表透過系統上傳處理。
- **表單擴展速度**：新增一種表單類型的平均時間 < 10 分鐘（從上傳範例到可判定）。

### 衡量方式
- 處理效率：系統記錄每次上傳的處理時間，與人工作業時間對照。
- 覆蓋率：統計已建立規格的 FormType 數量 vs 工廠實際使用的表單類型數量。
- 準確率：定期抽樣比對系統判定結果與人工複核結果。
- 採用率：統計系統上傳次數與歷史紀錄量。
- 擴展速度：記錄規格管理頁面的操作日誌。

## 7. 風險與假設

### 已知風險與應對方案
- **風險**：通用解析器無法處理所有 Excel 格式（約 10-15% 邊界情況）。
  - **應對**：持續擴展解析器能力（已支援標準行列、多層表頭、轉置格式）。對於無法處理的格式，提供手動建立規格的介面。
- **風險**：規格項目名稱與表頭文字不完全匹配，導致判定遺漏。
  - **應對**：已實作四層模糊匹配策略（精確 → 正規化 → 去單位 → 子字串），覆蓋大部分變體。
- **風險**：系統無身分驗證，正式部署有安全疑慮。
  - **應對**：在正式上線前，必須加入身分驗證層（JWT/Session）與角色權限控管。
- **風險**：DeepSeek API 不可用時影響表單辨識。
  - **應對**：AI 為可選功能，系統在無 AI 時自動 fallback 到規則引擎。6 級辨識的前 5 級不依賴 AI。
- **風險**：大量歷史資料導致查詢效能下降。
  - **應對**：已優化為 SQL 聚合查詢（2 條 SQL 取代 N+1），批次刪除使用 bulk delete。

### 關鍵假設
- 假設工廠的點檢表皆為 `.xlsx` 格式的 Excel 檔案。
- 假設大部分點檢表有中文表頭且為表格式佈局。
- 假設規格匯入後不會頻繁變更（版本追蹤為低頻操作）。
- 假設使用者具備基本的 Excel 操作能力。
- 假設部署環境有穩定的網路與 MySQL 資料庫。

### 資料模型概要

```
FormType ──< FormSpec ──< SpecItem
                 ├──< SpecVersion
UploadRecord ──< InspectionResult >── FormSpec
```

- **FormType** — 表單類型（代碼、名稱、辨識關鍵字、檔名模式、結構指紋）
- **FormSpec** — 設備規格群組（綁定表單類型 + 設備 ID）
- **SpecItem** — 規格項目（名稱、判定類型、上下限、預期值、群組）
- **SpecVersion** — 規格版本快照（版本號、來源、檔案雜湊、項目快照、變更摘要）
- **UploadRecord** — 上傳紀錄（檔名、批次 ID、狀態、表單類型）
- **InspectionResult** — 檢驗結果（原始數據、判定數據、整體結果）

### 解析器架構

| 解析器 | 覆蓋範圍 | 格式支援 |
|--------|----------|----------|
| 5 種內建解析器 | F-QA1021、F-RD09AA/AB/AJ/AK | 各表單專屬結構 |
| GenericParser — 標準模式 | 任意中文表頭表格 | 自動偵測表頭行 |
| GenericParser — 多層表頭 | 有子標題的表格 | 子標題合併為群組 |
| GenericParser — 轉置模式 | 日期橫向展開的表格 | 自動轉置為行列 |
| 表頭規格提取器 | 無匯總 Sheet 的檔案 | 從表頭自動提取規格 |

### 判定引擎匹配策略

```
1. 精確匹配：header_label == spec_item_name
2. 正規化匹配：全形→半形、去空白、小寫
3. 去單位匹配：「温度(℃)」→「温度」
4. 子字串包含：spec「温度」⊂ header「温度(℃)」
```
